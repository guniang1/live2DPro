# Live2D 模型包第一层资源规范化

## 需求

用户上传的 Live2D 包（如 VTube 导出）常与仓库标准包 **Xiaozi** 布局不一致：表情堆在根目录、换装类 exp 写入 model3、缺 Motions/HitAreas、LipSync 绑错参数、嵌套重复 `moca.*` 等，导致对话 LLM 误选服装切换项、点击无反应、口型异常。

第一层在**资源侧**将包整理为**仅含两个资源子目录**的标准布局，再 zip 上传或 `scan_package` 入库：

| 目录 | 内容 | 写入 model3 | 进入 catalog |
|------|------|-------------|--------------|
| `expressions/` | **全部** `.exp3.json`（含换装/道具类） | `Expressions` | 表情列表 |
| `motions/` | 仅 `.motion3.json` 身体动作 | `Motions` | 动作列表 |

**不存在** `wardrobe/` 或单数 `motion/` 目录；历史目录会在规范化时自动迁移。

## 处理逻辑

1. 读取 `.live2d-package.json`（无则按包名推断，**moka** 有内置模板）。
2. **扫描** `assess_package_standardness`：根目录 exp3、`motions/` 下误放 exp3、LipSync 绑眼、冗余 `textures/moca.*`、废弃目录等。
3. **上传 zip 固定执行**（无 `normalize` 开关、无环境变量关闭）：
   - 删除 `remove_paths` / `remove_globs` 冗余文件；
   - **全部** `.exp3.json` 移入 `expressions/` 并写入 model3 `Expressions`；
   - `.motion3.json` 移入 `motions/`；
   - 修补 LipSync、EyeBlink、可选 HitAreas（侧栏 `.live2d-hit-areas.json` 或 `data/hit_areas/{包名}.json`）。
4. 输出 `.live2d-normalize-report.json` 与 `normalize_warnings`。

### exp3 与 motion3

**不做** exp3 与 motion3 的混放：`motions/` 下仅保留 `.motion3.json`。

## 实际处理

| 项 | 路径 |
|----|------|
| 核心模块 | `PY/live2d_db/package_normalize.py` |
| HitAreas 模板 | `PY/live2d_db/hit_areas_templates.py` |
| CLI | `PY/scripts/normalize_live2d_package.py` |
| 示例配置 | `Demo/public/Resources/moka/.live2d-package.json` |

### 命令

```bash
cd PY
python scripts/normalize_live2d_package.py --package moka --init-config
python scripts/normalize_live2d_package.py --package moka --dry-run
python scripts/normalize_live2d_package.py --package moka
python -m live2d_db.scan_package --package moka --user-id 1
```

### 标准目录示例（moka）

```
moka/
  moka.model3.json
  moka.moc3
  textures/texture_00.png
  expressions/*.exp3.json    # 全部 exp3，写入 model3
  motions/*.motion3.json     # 若有身体动作
  .live2d-package.json
```

### 上传 API

`POST /api/live2d-model-assets/upload-zip`：**必定**先规范化再入库（无请求参数可关闭）。

**Form**：`user_id`、`model_zip`、可选 `package_key`。

**响应**：`normalized` 恒为 `true`（成功时）、`normalize_warnings`、`emotion_count`、`motion_count`、`hit_areas_source`。

## 前端对接

- 本地：`lappdefine.js` 的 `ModelDir` 含包名，入口 `{包名}.model3.json`。
- 远程：登录后 `ensureLive2dRemoteManifests`；`is_entry_model` 由上传索引写入。
- WebSocket **`catalog.expression`** 仅含 `expressions/` 下 exp3（且 model3 登记项）。
- **`catalog.motion`** 仅含 `.motion3.json` 标识名。
