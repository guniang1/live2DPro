# 架构与时序图

本文档给出与本仓库实现相对应的顺序图、流程图、分层组件图（UML 组件图风格）及 **UML 用例图**（Mermaid 绘制）。其中 **§1** 主对话顺序图对应论文 **§3.4 用例与交互流程**、**§3.6 数据流与时序**：键盘与语音仅在「得到用户 utterance 文本」之前分叉，**进入 `/ws/chat` 的载荷均为同一形态的文本消息**。可在 VS Code、GitHub、GitLab 等支持 Mermaid 的 Markdown 预览中渲染。

---

## 1. 顺序图：主对话端到端（键盘 / 语音汇合为文本）

**概念**：用户在界面上的输入路径有两种——直接键盘输入，或先经麦克风由 ASR 转为终稿字符串；二者在前端汇合为 **`sendChatMessage` → `/ws/chat`** 的 **`{ "message": "..." }`**，其后链路完全一致：**读 Redis/MySQL 记忆与人设 → Ollama 动作模型 → Ollama 聊天流式 → 服务端切段调用 TTS → 下行播放与口型**。

**与附带 Demo 一致的路径**：朗读帧默认经 **同一条 `/ws/chat`** 下发：先 JSON **`chunk_audio`** 元数据帧，**再紧随二进制 WAV**（合并流）；浏览器侧解码入队驱动播放与口型；**不依赖**独立朗读连接。字段语义以实现代码为准。

```mermaid
sequenceDiagram
    autonumber
    actor User as 用户
    participant FE as 浏览器客户端
    participant ASR as 语音识别网关（/ws/asr）
    participant Dash as 云端语音识别（DashScope）
    participant Chat as 对话编排服务（/ws/chat）
    participant Store as 缓存与持久化（Redis，MySQL）
    participant LLM as 大语言模型服务（Ollama）
    participant Eng as 语音合成服务（MiMo 或 GPT-SoVITS）

    alt 键盘输入
        User->>FE: 输入文本
    else 语音输入（识别后与键盘共用入口）
        User->>FE: 麦克风 PCM
        FE->>ASR: 建立 WebSocket（/ws/asr）
        ASR->>Dash: 建立实时识别会话
        loop PCM 帧上行
            FE->>ASR: 二进制 PCM
            Dash-->>ASR: 识别事件
            ASR-->>FE: 中间结果或终稿文本
        end
        FE->>FE: 确定终稿文本
    end

    FE->>Chat: WebSocket 文本帧（JSON，含用户消息字段）

    Chat->>Store: 读取分层记忆与人设等
    Chat->>LLM: 动作模型（非流式 JSON）
    LLM-->>Chat: 表情与动作字段等
    Chat->>LLM: 聊天补全（流式）

    par 文本下行（与朗读并行）
        loop 直至上游流结束
            LLM-->>Chat: 增量 token
            Chat-->>FE: 文本增量帧（首帧可含表情与动作）
        end
    and 朗读（切段、保序；默认合并流）
        loop 按序号合成下发
            Chat->>Eng: HTTP 请求合成当前句
            Eng-->>Chat: WAV 字节流
            Chat-->>FE: 朗读元数据帧及紧随的 WAV（同一连接）
            FE->>FE: 解码入队；音频能量驱动口型
        end
    end

    LLM-->>Chat: 生成流结束
    Chat-->>FE: 本轮结束标记
    Chat->>Store: 写入会话记录；更新瞬时记忆并挤出短期条目等
```

**配置提示**：ASR 依赖 **`DASHSCOPE_API_KEY`**；前端可通过 **`VITE_ASR_WS_URL`** 覆盖 **`/ws/asr`** 地址（见 `Demo/src/api/wsConfig.js`）。

## 2. 流程图：`/ws/chat` 一轮请求（逻辑主干）

```mermaid
flowchart TB
    START([收到 JSON message])
    START --> ACCEPT[已连接：下发 catalog]
    ACCEPT --> MEM[拼装 system：<br/>人设 + 短期 + 长期 Redis/MySQL]
    MEM --> ACT[线程池：动作 LLM<br/>解析 expression/motion]
    ACT --> STREAM[异步：聊天 LLM stream]
    subgraph BUF[流式缓冲]
        STREAM --> CHUNK{句末标点 /<br/>token 上限?}
        CHUNK -->|否| STREAM
        CHUNK -->|是| ENQ[sentence_queue<br/>分配 index]
    end
    ENQ --> WORKERS[TTS workers<br/>并行合成]
    WORKERS --> ORDER[_tts_flush_ordered<br/>按 index 发送]
    STREAM --> CHUNKTXT[/ws/chat：<br/>chunk 文本/]
    ORDER --> CHATOUT[/ws/chat：<br/>chunk_audio + WAV<br/>默认合并流/]
    STREAM --> DONE[type: done]
    DONE --> SAVE[(MySQL chat_session)]
    SAVE --> REDIS[(Redis 瞬时→挤出→短期)]
```

## 3. 分层架构图（UML 组件图风格）

`<<layer>>` 表示架构层；虚线箭头表示运行时依赖方向；连接器上标注主要协议。

```mermaid
flowchart TB
    subgraph L1["<<layer>> 浏览器层"]
        direction TB
        C_UI["组件: SPA Shell<br/>(index.html + main.js)"]
        C_L2D["组件: Live2D 运行时<br/>(Cubism Core / Framework + LApp*)"]
        C_NET["组件: 联网适配器<br/>(ws.js / apiBase / assetUpload / chatSessions)"]
        C_UI --> C_L2D
        C_UI --> C_NET
    end

    subgraph L2["<<layer>> 应用服务层"]
        direction TB
        C_API["组件: REST 网关<br/>live2d_db.http_api"]
        C_WS["组件: 实时网关<br/>router.wschat / router.asr_ws"]
        C_MEMJOB["组件: 长期记忆调度<br/>long_memory_consolidator"]
        C_MAIN["组件: FastAPI 宿主<br/>main.py + lifespan"]
        C_MAIN --> C_API
        C_MAIN --> C_WS
        C_MAIN --> C_MEMJOB
    end

    subgraph L3["<<layer>> 模型与媒体服务层"]
        direction TB
        C_OLLAMA["组件: 推理服务<br/>Ollama"]
        C_TTS["组件: 语音合成<br/>GPT-SoVITS / MiMo"]
        C_MINIO["组件: 对象存储<br/>MinIO / S3"]
        C_FFMPEG["组件: 媒体转码<br/>ffmpeg（TTS 参考音频等）"]
    end

    subgraph L4["<<layer>> 数据持久化层"]
        direction TB
        C_MYSQL["组件: 关系库<br/>MySQL"]
        C_REDIS["组件: 缓存与会话记忆<br/>Redis"]
    end

    C_NET -.->|HTTPS /api| C_API
    C_NET -.->|WSS /ws/chat、/ws/asr<br/>（Demo 默认；朗读走 chat 合并流）| C_WS

    C_WS -.->|HTTP 客户端| C_OLLAMA
    C_WS -.->|HTTP 客户端| C_TTS
    C_MEMJOB -.->|HTTP 客户端| C_OLLAMA

    C_API -.->|SQL| C_MYSQL
    C_WS -.->|SQL| C_MYSQL
    C_MEMJOB -.->|SQL| C_MYSQL

    C_WS -.->|Redis 协议| C_REDIS
    C_API -.->|Redis 协议| C_REDIS
    C_MEMJOB -.->|Redis 协议| C_REDIS

    C_API -.->|SDK / HTTP| C_MINIO

    C_API -.->|可选| C_FFMPEG
```

## 4. UML 用例图（核心用例）

参与者：**用户**（主要）；**定时扫描 / 提醒任务**（次要，对应库表 `remind_trigger` 等业务侧的待扫描逻辑，具体触发方式以实现为准）。

用例图中 **`«include»`** 表示语音路径在识别完成后 **包含** 将文本送入与文本对话相同的会话链路。

```mermaid
usecaseDiagram
    left to right direction

    actor "用户" as U
    actor "定时扫描与提醒任务" as S

    rectangle "Live2D 数字人系统" {
        usecase "登录 / 注销" as UC_Login
        usecase "文本对话" as UC_Text
        usecase "语音对话" as UC_Voice
        usecase "切换 Live2D 模型" as UC_Model
        usecase "接收主动关怀推送" as UC_Care
        usecase "查看历史会话" as UC_History
        usecase "上传模型音色与角色定义" as UC_AssetPersona
    }

    U --> UC_Login
    U --> UC_Text
    U --> UC_Voice
    U --> UC_Model
    U --> UC_Care
    U --> UC_History
    U --> UC_AssetPersona

    S --> UC_Care

    UC_Voice ..> UC_Text : «include»\n(ASR 结果进入对话)
```

**与实现的粗略对应（便于写论文「实现章节」对照）**

| 用例 | 典型前端 / 后端触点 |
|------|---------------------|
| 登录 / 注销 | `Login.html`、`live2d_info`、`POST /api/users`（登录分支） |
| 文本对话 | `ws.js` → `/ws/chat`（文本 `chunk`；朗读默认 `chunk_audio` + WAV 同连接） |
| 语音对话 | `SpeechRecognition.js`、`/ws/asr` → 终稿文本再走文本对话 |
| 切换 Live2D 模型 | 工具栏切换、`setLive2dPackage`、`reconnectChatWebSocketsForNewPackage` |
| 接收主动关怀推送 | `remind_trigger` + 业务推送通道（若当前仅为 REST 查询待触发项，可在正文注明「数据模型已具备，推送通道可扩展」） |
| 查看历史会话 | `chatSessions.js` → `GET /api/chat-sessions` |
| 上传模型 / 音色与角色定义 | `upload-zip`、`live2d-tts-refers/upload`、`characterDef.html` / `/personas/by-package` |
